操作説明書

クイックスタート

  1. カメラ開始: 「カメラ開始」ボタンをクリック
  2. アクセス許可: ブラウザのカメラアクセス許可を承認
  3. 検出開始: 「検出開始」ボタンで人物検出を開始
  4. 自動キャプチャ: 人物が検出されると自動的に画像保存

キーボードショートカット

docsドキュメント表示
Cカメラ開始/停止
Space検出開始/停止
S手動キャプチャ
?ヘルプ表示

設定項目

  • 自動キャプチャ: ON/OFF切り替え
  • キャプチャ間隔: 1〜10秒で設定
  • 検出信頼度: 0.3〜0.9で調整
    • 0.3〜0.5: 高感度（推奨）
    • 0.5〜0.6: バランス型
    • 0.7〜0.9: 高精度

画像管理

  • 個別削除: 各画像のゴミ箱アイコンをクリック
  • 選択機能: 画像をクリックして選択/選択解除
  • 一括保存: すべての画像を連続ダウンロード
  • 選択削除: 選択した画像のみ削除
  • 全削除: すべての画像を一括削除

要件定義書

プロジェクト概要

リアルタイムで人物を検出し、自動的に画像をキャプチャするWebアプリケーション。MediaPipe Poseを使用した高精度な人物検出機能を搭載。

機能要件

ID機能名説明優先度
FR-001カメラアクセスWebカメラのリアルタイム映像取得必須
FR-002人物検出MediaPipe Poseによる人物検出必須
FR-003自動キャプチャ検出時の自動画像保存必須
FR-004画像管理個別・一括削除、保存機能必須
FR-005設定管理検出閾値、間隔の調整重要
FR-006キーボード操作ショートカットキー対応重要
FR-007レスポンシブUIモバイル・デスクトップ対応重要
FR-008ドキュメント操作説明・技術資料重要
FR-009エラーハンドリング適切なエラー処理と表示重要
FR-010パフォーマンス30fps動作、低遅延重要

非機能要件

  • パフォーマンス: 30fps、検出遅延100ms以下
  • 精度: 人物検出精度90%以上
  • ブラウザ対応: Chrome90+, Edge90+, Firefox88+, Safari14+
  • システム要件: CPU: デュアルコア2.0GHz以上、RAM: 4GB以上
  • セキュリティ: HTTPS必須、カメラ権限適切処理

制約事項

  • HTTPS環境でのみカメラアクセス可能
  • 同時検出人数は3〜5人が上限
  • ブラウザメモリ制限により最大20枚の画像保持
  • File System APIは一部ブラウザで未対応
