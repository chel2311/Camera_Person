プロジェクト概要

リアルタイムで人物を検出し、自動的に画像をキャプチャするWebアプリケーション。MediaPipe Poseを使用した高精度な人物検出機能を搭載し、ユーザビリティと技術的完成度の両方を追求したプロフェッショナルなWebアプリです。

主要機能

リアルタイム検出

  • • MediaPipe Pose による高精度検出
  • • 30fps でのスムーズな動作
  • • 骨格追跡とキーポイント表示
  • • 信頼度による品質管理

自動キャプチャ

  • • 設定可能な間隔（1-10秒）
  • • 検出信頼度フィルタリング
  • • 手動キャプチャも対応
  • • 高品質な画像保存

画像管理

  • • 個別削除機能
  • • 一括選択・削除
  • • 連続ダウンロード
  • • ギャラリー表示

クイックガイド

5分で始める人物検出

このガイドに従うことで、わずか5分でアプリケーションを起動し、人物検出を開始できます。

ステップバイステップ

1
カメラアクセス許可
「カメラ開始」ボタンをクリック → ブラウザの許可ダイアログで「許可」を選択
2
検出開始
「検出開始」ボタンをクリック → リアルタイムで人物の骨格が表示される
3
設定調整
検出信頼度を0.3〜0.5に設定（推奨）→ キャプチャ間隔を調整
4
自動キャプチャ
人物が検出されると自動的に画像が保存される → ギャラリーで確認

推奨設定

高感度モード

  • • 検出信頼度: 0.3-0.4
  • • キャプチャ間隔: 2-3秒
  • • 用途: 一般的な検出

バランスモード

  • • 検出信頼度: 0.5-0.6
  • • キャプチャ間隔: 3-5秒
  • • 用途: 標準的な利用

高精度モード

  • • 検出信頼度: 0.7-0.9
  • • キャプチャ間隔: 5-8秒
  • • 用途: 精密な検出が必要

トラブルシューティング

よくある問題と解決策

カメラが起動しない

症状: 「カメラ開始」ボタンを押してもカメラが起動しない

原因と対策:

  • HTTPS環境: HTTPSでアクセスしているか確認（必須要件）
  • ブラウザ許可: カメラアクセス許可が拒否されていないか確認
  • カメラ占有: 他のアプリケーションがカメラを使用していないか確認
  • ドライバー: カメラドライバーが正常にインストールされているか確認

人物が検出されない

症状: カメラは動作するが人物が検出されない

推奨解決手順:

  1. 信頼度調整: 検出信頼度を0.3-0.4に下げる
  2. 距離調整: カメラから1-3mの距離に立つ
  3. 照明改善: 十分な明るさを確保
  4. 背景考慮: 複雑な背景を避ける
  5. 全身表示: 上半身だけでなく全身をフレームに入れる

キャプチャされない

症状: 人物は検出されるがキャプチャが実行されない

チェックポイント:

  • 自動キャプチャ: 設定でONになっているか確認
  • 間隔設定: キャプチャ間隔が適切か確認（推奨：2-4秒）
  • 信頼度: 検出信頼度が高すぎないか確認
  • 検出持続: 十分な時間検出状態を維持しているか

パフォーマンス問題

エラーコード対応表

エラーコードエラー名原因対処法
NotAllowedError アクセス拒否 カメラ許可が拒否された ブラウザ設定でカメラアクセスを許可
NotFoundError デバイス未検出 カメラが見つからない カメラ接続確認、ドライバー再インストール
NotReadableError 読み取り不可 カメラが他で使用中 他のアプリケーションを終了
OverconstrainedError 制約エラー 要求仕様がサポート外 カメラ解像度設定を下げる
MediaPipe Load Error AI読み込み失敗 ネットワーク問題 ページリロード、ネットワーク確認
